2. 코드스페이스에 quarto 설치

- 아래를 실행 (하면 안될걸?)

dpkg -i quarto-1.9.37-linux-amd64.deb

3. 깃헙추가기능1: 블로그 업로드

- 깃헙과 구글드라이브의 두번째 차이: 깃헙은 단순한 저장소를 넘어 추가기능이 있음. 구체적으로 GitHub 레포지토리는 블로그 게시물을 자동으로 발행(publish)할 수 있는 기능을 지원함.

- 코드스페이스에 quarto 설치 에서 명령어를 수정하고 (dpkg → sudo dpkg) 업로드하자.

quarto render
git add .
git commit -m .
git push

- 깃헙에서 아래를 설정 – /docs 옆에 Save를 눌러야함

- 꾸미기: 아래를 수정하자..

4. .sh

- 블로그에 업로드할때마다..

quarto render
git add .
git commit -m .
git push

하는것 귀찮죠?

- 자동화하는 방법.

  1. publish.sh 파일을 만든다.
  2. quarto render, git add ., git commit -m ., git push 를 순서대로 친다.
  3. 저장한다.
  4. 터미널에서 bash publish.sh 실행

6. 깃헙추가기능2: 패키지 배포

- 깃헙과 구글드라이브의 두번째 차이: 깃헙은 단순한 저장소를 넘어 추가기능이 있음. 예를 들어, GitHub 레포지토리는 소프트웨어 패키지를 배포하는 데 활용될 수 있음.

아래의 내용은 가급적 터미널로 해보세요

A. 저장소 만들기

  • 저장소 이름은 asdf 로 하자. (대충 만들겠다는 의미)

B. 저장소에서 대충 아래와 같은 구조로 파일 및 디렉터리를 만든다

asdf/                  # GitHub 저장소 이름
│
├── pkg0506/           # Python에서 import할 때 사용하는 이름
│   └── __init__.py    # 반드시 이 이름으로 작성해야 함
│
├── README.md          # 선택 사항 (없어도 무방)
├── pyproject.toml     # 패키지 설치 및 배포를 정의하는 파일로 반드시 이 이름이어야 함

– __init__.py 파일내용

asdf/pkg0506/__init__.py
def hello_world():
    return "Hello from pkg0506!"

– pyproject.toml 파일내용

asdf/pyproject.toml
[build-system]                              # 빌드 방법을 pip에게 알려주는 부분 (거의 고정)
requires = ["setuptools>=61.0"]             # 빌드에 필요한 도구
build-backend = "setuptools.build_meta"     # 실제로 빌드하는 백엔드

[project]
name = "mypkg"        # pip install 할 때 사용되는 이름 (배포용 이름)
version = "0.1.0"     # 패키지 버전

C. 반영사항을 저장소로 push 한다

git add .
git commit -m .
git push

D. 코랩 등 다른 환경으로 가서 방금 만든 패키지를 설치한다

pip install git+https://github.com/miruetoto/asdf.git

E. 테스트한다

import pkg0506
pkg0506.hello_world()

F. 설치되어 있는지 확인한다

pip list | grep mypkg

mypkg 는 pyproject.toml 의 name= 에 적은 배포용 이름임. (import pkg0506 의 pkg0506 과 다름에 주의)

G. 삭제한다

pip uninstall mypkg
