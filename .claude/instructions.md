# 프로젝트 작업 규칙

## 중요: 폴더 구조와 작업 원칙

### 1. 메인 작업 공간
- **posts/** - 주 작업 폴더, 모든 강의 자료와 퀴즈는 여기에 작성
  - 예: Quiz4.qmd, 06wk.qmd 등

### 2. 참고 자료 (절대 수정 금지)
- **pp2024/** - 2024년 참고 자료 (읽기 전용)
- **dstbx2025/** - 2025년 참고 자료 (읽기 전용)

**⚠️ 경고: pp2024/, dstbx2025/ 폴더의 파일은 절대 수정하거나 삭제하지 말 것!**

## 퀴즈 작성 규칙

### Quiz 파일 생성 시:
1. **위치**: 항상 `posts/` 폴더에 생성
2. **참고 자료**: pp2024/의 강의 노트 참조 (04wk-2.ipynb, 05wk-2.ipynb 등)
3. **포맷**: Quiz3.ipynb의 형식을 따를 것
4. **내용**: 참고 자료의 실제 강의 내용을 기반으로 문제 출제

### ⚠️ 중요: 퀴즈 작성 워크플로우
1. **1단계**: 항상 `.qmd` 파일로 먼저 작성 (예: `posts/Quiz4.qmd`)
2. **2단계**: 사용자 컨펌 대기
3. **3단계**: 컨펌 후 최종적으로 `.ipynb`로 변환 (예: `posts/Quiz4.ipynb`)

**절대 처음부터 .ipynb로 만들지 말 것! 반드시 .qmd → 컨펌 → .ipynb 순서를 따를 것**

### 예시:
- Quiz4 작성 시:
  - 읽기: `pp2024/04wk-2.ipynb`, `pp2024/05wk-2.ipynb` (참고용)
  - 쓰기: `posts/Quiz4.qmd` (1차 작성) → 컨펌 → `posts/Quiz4.ipynb` (최종)
  - 금지: `pp2024/Quiz4.ipynb` 생성 ❌

## 작업 흐름
```
pp2024/ (읽기) → 내용 분석 → posts/ (쓰기)
dstbx2025/ (읽기) → 내용 분석 → posts/ (쓰기)
```
